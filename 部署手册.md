## 🚀 一键部署脚本

### 方法1: 使用rsync (推荐)

```bash
# 1. 在本地执行 (Windows PowerShell)
# 安装rsync (如果未安装)
# choco install rsync

# 2. 同步后端代码到服务器
rsync -avz --exclude='node_modules' --exclude='.env' ^
  "I:\qishouxiangmu11111\kuailv\backend\*" ^
  root@[服务器IP]:/var/www/kuailv/backend/

# 3. SSH到服务器并部署
ssh root@[服务器IP] << 'EOF'
cd /var/www/kuailv/backend

# 安装依赖
npm install --production

# 重启服务
pm2 restart yanfu-backend

# 查看状态
pm2 status
pm2 logs yanfu-backend --lines 50
EOF
```

### 方法2: 使用scp (简单文件上传)

```bash
# 1. 上传核心文件
scp app.js root@[服务器IP]:/var/www/kuailv/backend/
scp package.json root@[服务器IP]:/var/www/kuailv/backend/

# 2. 上传路由文件
scp -r routes/* root@[服务器IP]:/var/www/kuailv/backend/routes/

# 3. 上传服务文件
scp -r services/* root@[服务器IP]:/var/www/kuailv/backend/services/

# 4. SSH到服务器重启
ssh root@[服务器IP] "cd /var/www/kuailv/backend && pm2 restart yanfu-backend"
```

### 方法3: 使用Git部署

```bash
# 1. 在服务器上克隆代码
ssh root@[服务器IP] << 'EOF'
cd /var/www/kuailv
git pull origin main  # 或 master

# 安装依赖
cd backend
npm install --production

# 重启服务
pm2 restart yanfu-backend
pm2 logs yanfu-backend --lines 50
EOF
```

## 🐛 常见问题排查

### 问题1: 服务无法启动

```bash
# 查看错误日志
pm2 logs yanfu-backend --err --lines 100

# 检查端口占用
netstat -tlnp | grep 3001

# 检查Node.js版本
node -v

# 重新安装依赖
cd /var/www/kuailv/backend
rm -rf node_modules
npm install
```

### 问题2: 数据库连接失败

```bash
# 检查MySQL状态
systemctl status mysql

# 测试连接
mysql -u root -p -e "SELECT 1"

# 检查数据库是否存在
mysql -u root -p -e "SHOW DATABASES;"
```

### 问题3: Nginx代理失败

```bash
# 检查Nginx配置
cat /etc/nginx/sites-available/kuailv

# 测试配置
nginx -t

# 查看错误日志
tail -f /var/log/nginx/error.log
```

### 问题4: CORS错误

检查 `.env` 文件中的 `ALLOWED_ORIGINS` 配置：

```bash
cat /var/www/kuailv/backend/.env
```
